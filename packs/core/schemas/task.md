# Schema: task

A **Task** is a first-class note representing one discrete commitment. One note per task (not inline checkboxes).

File path: `Ops/Tasks/<title slug>.md`

## Frontmatter

```yaml
---
type: task
id: task-YYYYMMDD-NNN
title: "<concise imperative>"
status: next              # inbox | backlog | next | scheduled | in_progress | waiting | needs_review | done | canceled
project: "[[<Name>]]"   # may be empty if task is area-level
area: "[[<Name>]]"
owner: me                          # me | "[[<Name>]]" | "[[Agent - <role>]]"
priority: p2              # p0 | p1 | p2 | p3
importance: 4             # 1–5
urgency: 3                # 1–5
effort: 45m               # human-readable duration: 5m, 25m, 45m, 1h, 2h, 1d
actual_effort:            # set on close; same format as effort. May be triangulated by observe-task-actuals.
actual_effort_source:     # self-report | email-timestamp | doc-mtime | git-log | calendar-block | unknown
energy: medium            # low | medium | high
context:                  # tags that constrain where/when
  - computer
  - deep-work
due: YYYY-MM-DD
scheduled_start:          # ISO datetime if time-blocked
scheduled_end:
follow_up_date:
blocked_by: []
unblocks: []
# People relationships — keep these distinct
people:                   # everyone touched by this task
  - "[[<Name>]]"
requested_by:
assigned_to:              # who is doing it (if not `owner`)
waiting_on:               # who or what is blocking
delegate_candidate:       # who could take this off your plate
reviewer:                 # who must approve before done
stakeholders: []          # who should be kept in the loop
# Agent delegation
delegate_type: human      # me | human | agent
agent_eligible: false
agent_type:               # planning | research | writing | coding | admin | data-analysis
human_review_required: true
acceptance_criteria:
  - "<criterion 1>"
sensitivity: normal
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

## Body sections

- `# Objective` — one sentence
- `# Context` — what you need to know to start
- `# Inputs` — wiki-linked sources, project page, decisions
- `# Steps` — bullet list (optional)
- `# Acceptance criteria` — must match frontmatter
- `# Risks / constraints`
- `# Starter prompt` — generated by the planner agent; ready to paste into a fresh LLM session
- `# Agent handoff` — present only if `agent_eligible: true`; structured per the agent-handoff template
- `# Work log` — append-only; each work session adds a dated entry

## Rules

- A task without a project is permitted only if `area:` is set. The auditor flags tasks with neither.
- `status: waiting` requires `waiting_on:` to be populated.
- `status: scheduled` requires `scheduled_start:` and `scheduled_end:`.
- `status: done` requires a final entry in `# Work log` with the outcome.
- `agent_eligible: true` requires complete `acceptance_criteria` and a populated `# Agent handoff` section.
- `actual_effort:` is optional but encouraged when known. If set, `actual_effort_source:` should record how the duration was derived (self-report or which observable signal). The `observe-task-actuals` skill populates these on closed Tasks where it can triangulate a signal, and surfaces remaining Tasks for sparse self-report at the weekly review.
